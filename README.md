# 🌌 Hire-Up | My Universe (NgHighchartsCanvas)

Современная экосистема для продуктивности, карьерного роста и непрерывного обучения ИТ-разработчика. Проект представляет собой архитектурно сложную SPA-платформу, объединяющую интеллектуальные инструменты автоматического аудита документов (резюме/договоров), интерактивную базу знаний для подготовки к техническим собеседованиям и модули тестирования.

Вся аналитика востребованности навыков и параметров контрактов визуализируется на лету с помощью кастомных интерактивных графиков, превращая сухие данные в наглядную карту карьерного развития.

---

## 🎯 Архитектурная ценность и стек

Проект спроектирован по канонам промышленной разработки крупномасштабных приложений. Главная ценность — демонстрация продвинутого владения архитектурой **Nx Monorepo**, современным **Angular 18+**, гибридным стейт-менеджментом и интерактивной визуализацией данных.

### Ключевые технологии и версии:

- **Монорепозиторий:** `Nx v20.0.0` (Project Crystal, автоматическое управление графом зависимостей)
- **Фреймворк:** `Angular v18.2.0` (Standalone-архитектура, Строго типизированные формы, Signal-компоненты)
- **Визуализация данных:** `Highcharts` / `highcharts-angular` (Интерактивные графики аналитики рынка)
- **Стиль и UI:** `Angular Material v18.2.14` + `SCSS` (Кастомная UI-библиотека)
- **Глобальный стейт:** `@ngxs/store v18.1.5` (Паттерн Command/Event на классах и декораторах для сквозных процессов)
- **Локальный стейт:** `@ngrx/signals v18.0.0` (Новейший функциональный Signal Store для реактивного управления модулями скрининга)
- **Среда тестирования:** `Jest` (Unit-тесты) + `Cypress` (E2E-тесты)

## 🏗️ Микрофронтенд-архитектура (Граф Nx)

Код разделен на независимые изолированные слои, что исключает циклические зависимости, гарантирует строгую инкапсуляцию бизнес-логики и масштабируемость:

```text
apps/
  └── hire-up/            # Приложение-оболочка (Shell). Предоставляет глобальный Layout и роутинг.
libs/
  ├── interview-tools/    # Feature-либа: Инструменты ИТ-ассистента карьеры (Скринеры резюме и договоров)
  │     ├── main-dashboard/     # Главная страница (умный контейнер управления табами)
  │     ├── resume-screener/    # Модуль скрининга резюме (справочники рынка, UI, сигналы)
  │     └── contract-screener/  # Модуль скрининга трудовых договоров и офферов (юридический аудит)
  ├── knowledge-base/     # Feature-либа: База знаний и интерактивные тесты для обучения (Training)
  ├── shared/             # Data-Access либа: Общий сервис ItemsService, Firebase REST API, модели данных
  └── ui/                 # UI-Kit либа: Презентационные Dumb-компоненты (Sidebar, Header, графики Highcharts)

```

## ⚡ Функциональные и архитектурные особенности

1. **Гибридный стейт-менеджмент:** Управление глобальными критическими действиями приложения через **NGXS (Redux)**, и легковесное, реактивное управление локальными данными скрининга и аналитики через **NgRx Signal Store** и нативные Angular Signals.
2. **Продвинутая дата-визуализация (Highcharts):** Интеграция графиков для интерактивного отображения частоты встречаемости навыков (Топ-40 навыков) и ключевых слов в опыте работы (Топ-69 слов) относительно общего объема рынка вакансий.
3. **Декларативные справочники аудита:** Бизнес-логика валидации вынесена в изолированные типизированные структуры данных (`Record<string, Definition>`). Каждому критерию сопоставлена своя степень критичности (`critical` / `minor`), на основе чего динамически рассчитывается итоговый "Индекс безопасности" договора или качество резюме.
4. **Универсальный Data-Access слой:** Базовый сервис `ItemsService` инкапсулирует абстрактную логику обработки данных и автоматического маппинга структур (трансформация Object-to-Array для Firebase RTDB и поддержка REST для JSON Server).
5. **Строгая инкапсуляция (DI):** Библиотеки не зависят от окружения приложений. Конкретные API эндпоинты поставляются в либы "на лету" через механизмы `InjectionToken`.
6. **Strict Typed Forms:** Формы добавления контента и ввода текстов для анализа написаны с использованием `NonNullableFormBuilder` и жестких интерфейсов, исключающих появление `null` на этапе компиляции.
7. **Route-Scoped Stores:** Сторы инициализируются в конфигурации роутера, живут ровно столько, сколько открыта вкладка конкретного скринера, и автоматически уничтожаются, исключая утечки памяти.
8. **Smart/Dumb паттерн:** Четкое разделение компонентов на презентационные (принимают сигналы через `input.required()` и отображают иконки/тексты в зависимости от `UiStateConfig`) и контейнерные.

## 🌍 CI/CD и синхронизация данных

Проект настроен на бесшовную работу в двух режимах (Dev/Prod) через автоматическую подмену конфигураций Nx:

- **Локальная разработка:** Фронтенд общается с локальным `json-server` (Mocks БД в файле `db.json`) через настроенный `proxy.conf.json`.
- **Продакшен (GitHub Pages):** Настроен автоматический пайплайн через GitHub Actions. При сборке конфигурация переключается на **Firebase Realtime Database REST API**, обеспечивая полноценное сохранение, чтение и модификацию данных в облаке прямо со статического хостинга.
